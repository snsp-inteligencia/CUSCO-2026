CUSCO 2026 v1.3.6 — FOLIO OPTIMIZADO

CAMBIO PRINCIPAL
El consecutivo ya no recorre toda la columna Folio en cada registro.

Ahora usa:
PropertiesService.getScriptProperties()
Propiedad: CUSCO_LAST_FOLIO_NUMBER

VENTAJAS
- Mayor velocidad cuando existan miles de encuestas.
- LockService impide que dos dispositivos reciban el mismo folio.
- El folio conserva el formato CUSCO-QRO-00001.
- La hoja solo se revisa la primera vez que se inicializa el contador.

EXCEL / GOOGLE SHEETS
En la hoja REGISTROS cambia manualmente:
No. -> Folio
ID  -> CURP
No alteres el orden de las columnas.

PRIMERA INSTALACIÓN
1. Sustituye el código del proyecto por CODE_APPS_SCRIPT_V1_3_6.gs.
2. Guarda el proyecto.
3. En el selector de funciones elige:
   sincronizarConsecutivoDesdeHoja
4. Presiona Ejecutar una sola vez y autoriza el proyecto.
5. Revisa el registro de ejecución:
   - si la hoja está vacía, el siguiente será CUSCO-QRO-00001;
   - si ya existe CUSCO-QRO-00001, el siguiente será CUSCO-QRO-00002.
6. Crea una nueva implementación como Aplicación web.
7. Ejecutar como: tú.
8. Acceso: cualquier usuario con el enlace.
9. Si cambia la URL /exec, reemplázala en APPS_SCRIPT_URL dentro de index.html.
10. Sustituye en GitHub index.html, manifest.json y service-worker.js.

CUÁNDO VOLVER A EJECUTAR sincronizarConsecutivoDesdeHoja
Solo cuando:
- se borren o cambien folios manualmente;
- se sustituya la hoja REGISTROS;
- se importe una base anterior;
- se desee reiniciar una prueba.

IMPORTANTE
No cambies directamente el valor de CUSCO_LAST_FOLIO_NUMBER salvo que sepas
exactamente qué consecutivo debe seguir.
